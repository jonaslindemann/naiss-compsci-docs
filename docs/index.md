# NAISS Development Best-practice Guides

!!! info
    This is a living document and needs more improvements before going into user service. It is a first sketch of best-practice site. Please contribute suggestions and corrections to help improve the guides for everyone.

Welcome to the NAISS Development best-practice guides! This comprehensive documentation provides best practices, tutorials, and reference materials for software development on NAISS HPC systems.

## Overview

These guides are designed to help researchers and developers effectively utilize NAISS (National Academic Infrastructure for Supercomputing in Sweden) high-performance computing resources. Whether you're compiling code, managing dependencies, or optimizing performance, you'll find practical guidance here.

## What You'll Find

- **[Compilers](compilers/prgenv.md)**: Configure compiler environments and optimize your code
- **[Build Systems](build-systems/cmake.md)**: Set up and troubleshoot build systems like CMake
- **[Profiling & Debugging](profiling-debugging/debugging.md)**: Debug and profile your applications
- **[Python](python/environments.md)**: Create isolated Python environments and optimize performance
- **[Containers](containers/apptainer.md)**: Use Apptainer for containerized applications
- **[Portability](portability/naiss-systems.md)**: Write portable code across NAISS systems

## Contributing

These guides are continuously updated to reflect best practices and new developments. If you have suggestions or corrections, please contribute back to the community.
